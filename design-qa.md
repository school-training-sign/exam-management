## Issues and fixes

### P1 — 모바일 필터와 상단 탭의 가로 넘침

- 증거: `qa/prototype/absence-mobile-before-fix.jpg`
- 원인: 셀렉트의 고유 최소 너비와 모바일 그리드의 `1fr` 최소 크기가 390px 프레임보다 커졌다.
- 수정:
  - 입력 요소에 `min-width: 0`
  - 필터를 `repeat(2, minmax(0, 1fr))`로 변경
  - 마지막 홀수 필터는 한 줄 전체 사용
  - 상단 탭을 `flex: 1 1 25%`와 `min-width: 0`으로 변경
- 사후 확인: 프로덕션 CSS 번들 생성, 390px 전용 규칙 정적 검사, 빌드 통과.

### P1 — 세션 복구 시 첫 학급이 선택되지 않음

- 증거: 모바일 세션 시드에서 학생 목록이 빈 상태로 표시됨.
- 원인: 화면이 빈 부트스트랩으로 먼저 마운트된 뒤 데이터를 받아도 `examDate`와 `classId` 초기 상태가 갱신되지 않았다.
- 수정: 결시·고사본부·자리배치·설정 화면에 부트스트랩 도착 후 첫 날짜·학급·학생을 선택하는 효과를 추가했다.
- 사후 확인: 단위 테스트와 프로덕션 빌드 통과.

### P2 — 전체 페이지 캡처에서 고정 헤더가 반복됨

- 원인: 인앱 브라우저의 전체 페이지 스티칭과 `position: sticky` 조합.
- 수정: 최종 시각 비교는 상단 고정 헤더가 한 번만 포함되는 뷰포트 캡처를 사용했다. 실제 페이지 렌더링 문제는 아니었다.

### P2 — 자리배치 순서가 한 방향으로만 고정됨

- 원인: 좌석 생성 함수가 행을 먼저 순회해 가로 순번만 만들었다.
- 수정: 가로 순번(행 우선)과 세로 순번(열 우선)을 선택하게 하고, 미리보기·저장·불러오기·인쇄 데이터에 순번과 방식을 함께 보존했다.
- 사후 확인: 6행×5열 세로 순번이 첫 행 `1, 7, 13, 19, 25`로 표시되고, 저장 후 다시 불러와도 `column` 값과 번호 배열이 유지됐다. 390×844에서 문서 가로 넘침은 없었다.

### P1 — 모바일 고사명 입력이 불필요하게 길어짐

- 원인: 데스크톱에서 고사명 칸을 넓히는 `flex-basis`가 모바일 세로형 폼에서는 높이로 적용됐다.
- 수정: 560px 이하에서 고사명 라벨의 flex 크기를 해제했다.
- 사후 확인: 390×844에서 고사명 입력 높이 42px, 문서 `scrollWidth`와 `clientWidth`가 모두 375px로 확인됐다.

### P1 — 비밀 링크 조회가 미사용 만료를 계속 연장함

- 원인: 30초 자동 갱신 조회가 링크 세션의 마지막 활동 시각을 갱신했다.
- 수정: 수동 저장과 실제 사용자 활동 신호만 30분 세션을 연장하고, 자동 조회는 세션을 연장하지 않게 분리했다.
- 사후 확인: 잘못된 토큰은 전용 오류 화면에서 차단되고, 정상 토큰은 교환 직후 URL fragment가 제거됐다.

### P1 — 포장 설정의 쉬는 시간 담당자를 입력할 수 없음

- 원인: 포장 과목 배정 불가 규칙을 담당자 입력에도 잘못 적용했다.
- 수정: 쉬는 시간에는 과목만 배정하지 못하게 하고, 날짜·시간대별 교무 담당자는 모든 행에 입력할 수 있게 했다.
- 사후 확인: 관리자 설정의 2개 포장일×4개 시간행에 담당자 입력 8개가 표시됐다.
