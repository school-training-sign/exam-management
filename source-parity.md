# 참고 서비스 기능 대조

2026-07-20에 참고 서비스의 운영 흐름을 가상 학급·가상 학생·임시 고사 정보로 확인했다. 확인 과정에서 만든 자료는 기능별 조사가 끝난 뒤 역순으로 삭제 대상으로 관리하며, 최종 잔존 여부를 별도 원격 조사 세션에서 재확인한다. 캡처와 내려받은 파일은 비공개 로컬 QA 자료로만 보관한다. 참고 서비스의 코드·이미지·제작자 표시·기존 이용자 데이터는 이 저장소에 포함하지 않는다.

## 확인한 흐름

### 결시 입력

- 고사일 → 학급 → 교시를 선택한 뒤 `불러오기`를 눌러 명단을 연다.
- 학생은 화면에서 결시 여부를 먼저 선택하고 사유를 `인정`, `미인정`, `질병`, `기타` 중에서 고른다.
- 새 결시자의 기본 사유는 `미인정`이다.
- `기타`를 고르면 상세 사유가 필수로 열린다.
- 변경 내용은 학생별로 즉시 저장하지 않는다. 전체 명단을 확인한 뒤 한 번에 제출한다.
- 제출 뒤에도 같은 화면에서 수정할 수 있으며 다시 제출하면 해당 학급·교시의 결시 명단이 교체된다.

### 고사본부

- 고사일·학년·학급·교시 기준으로 재적, 응시, 결시, 사유별 인원과 학급별 제출 상태를 함께 확인한다.
- 기간 조회는 시작일·종료일·학년·학급을 적용하고 결시 상세, 사유별, 학급별, 일자별, 학생별 누계를 보여준다.
- 현재 교시 현황과 기간 합산 통계는 서로 다른 Excel 결과로 내려받는다.
- 화면이 보이는 동안에만 30초마다 갱신한다.

### 시간표와 선택과목

- 같은 교시에 공통과목 또는 여러 선택과목을 둘 수 있다.
- 공통과목은 실제 적용 학급을 선택하며, 선택과목은 학생별 과목과 호실을 등록한다.
- 학생별 호실이 비어 있으면 해당 시간표에 지정한 기본 호실을 사용한다.
- 개인 시간표는 학년 전체 또는 특정 학급을 선택해 A4 학생 카드로 일괄 출력한다.
- 선택과목이 없는 시간은 개인 시간표에서 `자습`으로 표시한다.

### 자리배치

- 기본 좌석은 6행×5열이며 창가에서 복도 방향으로 배치한다.
- 별실 모드는 학년 전체 선택과목 응시자를 과목·호실별로 나눠 생성한다.
- 각자교실 모드는 해당 학급 전체를 유지하되 응시자와 결시자를 먼저, 미응시자를 뒤에 배치한다.
- 결시자는 좌석을 유지하면서 결시 표시를 남긴다.
- 제외 좌석을 편집하고 저장한 배치를 다시 불러오거나 삭제할 수 있다.
- 같은 조건의 배치를 다시 저장하면 새 항목을 중복 생성하지 않고 기존 항목을 갱신한다.
- 인쇄물에는 과목명, 호실, 칠판, 창가, 복도, 결시 범례가 포함된다.

### 빈 상태와 삭제

- 등록 자료가 없으면 다음 작업을 안내하는 빈 상태를 표시한다.
- 삭제 전에는 영향받는 학생·시간표·선택과목·결시·제출·자리배치 건수를 알리고 확인을 받는다.
- 학생 삭제는 과거 기록 연결을 보존하기 위해 비활성화로 처리한다.
- 학급·고사일처럼 참조 범위가 큰 항목은 연결 자료를 함께 정리한다.

## 의도적으로 다르게 구현한 부분

- 참고 서비스의 좁은 화면 탭 가로 넘침은 재현하지 않고 390px에서도 한 열로 읽히게 했다.
- 참고 서비스의 색감과 정보 계층만 참고하고 컴포넌트, 문구, 아이콘, 코드는 독립적으로 구성했다.
- 고사본부·설정·자리배치·삭제·내보내기는 별도 관리자 세션으로 보호한다.
- 결시 일괄 제출에는 revision 충돌 검사를 추가해 동시에 연 화면이 최신 상태를 덮어쓰지 못하게 한다.
- 학교코드와 관리자 암호는 프런트엔드나 시트 셀에 원문으로 두지 않는다.
- 실제 데이터가 없는 `?demo=1`은 운영 API로 쓰기 요청을 보내지 않는다.

## 공개 저장소 제외 항목

- 참고 서비스 화면 캡처와 내려받은 출력물
- 조사에 쓴 가상 학생·가상 과목·임시 고사 정보
- 학교코드, 관리자 암호, 세션 토큰, Apps Script 배포 전용 정보
- 학생 실명과 실제 운영 데이터
